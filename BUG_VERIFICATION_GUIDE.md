# Bug 验证指南

## 测试场景

**请按以下步骤在真机上测试：**

### 步骤1：启动应用并进入练习模式

### 步骤2：设置测试场景
- 密码会是随机生成的，但我们假设它是：`红、黄、绿、蓝`
- 如果不是这4个颜色，请重新开始游戏直到获得这个组合

### 步骤3：提交猜测
填入颜色：
- 位置0：**红色** ✅（应该匹配）
- 位置1：**蓝色**
- 位置2：**黄色**
- 位置3：**绿色**

点击"确认"提交

### 步骤4：检查提示

**预期结果：**
- 1个绿色圆点（红色位置正确）
- 3个黄色圆点（蓝色、黄色、绿色颜色正确但位置错）

**如果实际结果不同，请记录：**
1. 显示了几个绿色？
2. 显示了几个黄色？
3. 显示了几个灰色？

### 步骤5：查看 Hilog 日志

在 DevEco Studio 的 Hilog 窗口搜索：
```
=== 评估猜测 ===
```

您应该看到类似：
```
=== 评估猜测 ===
密码: red, yellow, green, blue
猜测: red, blue, yellow, green
命中: 1
剩余猜测: blue, yellow, green
剩余密码: yellow, green, blue
  匹配颜色 blue 在位置 2, 伪命中累计: 1
  匹配颜色 yellow 在位置 0, 伪命中累计: 2
  匹配颜色 green 在位置 1, 伪命中累计: 3
伪命中: 3
================
```

## 如果问题仍然存在

**请告诉我：**
1. Hilog 日志中的完整输出
2. 屏幕上实际显示的提示圆点颜色和数量
3. 您填入的具体颜色和顺序

这样我就能准确定位问题所在！
